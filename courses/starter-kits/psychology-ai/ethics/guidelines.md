# Ethical Guidelines for Psychology + AI

> AI Guidance, Not AI Therapy

## Core Principle

AI is a mirror, not a therapist. It reflects patterns — it doesn't treat conditions.

## What AI CAN Do

### ✅ Appropriate Uses

| Use Case | Description | Example |
|----------|-------------|---------|
| **Pattern tracking** | Record and identify patterns | "You mentioned feeling tired 5 times this week" |
| **Data summarization** | Condense information | Session note summaries |
| **Insight surfacing** | Show correlations | "Sleep and mood seem connected" |
| **Research support** | Analyze research data | Thematic analysis |
| **Administrative** | Note-taking, scheduling | Session scheduling |

### ✅ Safe Practices

1. **Transparency**: Always tell clients when AI is involved
2. **Consent**: Get informed consent for AI analysis
3. **Data ownership**: Client owns their data
4. **Local processing**: Prefer local over cloud when possible
5. **Human review**: AI output reviewed by professional

## What AI CANNOT Do

### ❌ Prohibited Uses

| Prohibited | Why | Alternative |
|------------|-----|-------------|
| **Diagnosis** | Requires clinical judgment | AI can summarize; human diagnoses |
| **Therapy** | Requires human relationship | AI supports; human provides therapy |
| **Crisis intervention** | Lives at stake | Always human intervention |
| **Medication advice** | Medical domain | Refer to medical professional |
| **Final decisions** | Requires human judgment | AI informs; human decides |

### ❌ Red Lines

1. **Never let AI diagnose** — summarize symptoms, don't label
2. **Never let AI prescribe** — information only, not treatment
3. **Never remove human oversight** — AI assists, never replaces
4. **Never ignore consent** — always get explicit permission
5. **Never trust AI for crisis** — human response required

## The Non-Clinical Position

> "AI Guidance, Not AI Therapy"

| Guidance | Therapy |
|----------|---------|
| Direction, information | Treatment, intervention |
| Support, tools | Healing, relationship |
| AI appropriate | Human required |

**AI stays in guidance lane.**

## Consent Framework

### Required Disclosure

Tell clients:
1. AI will analyze their data
2. What AI will/won't do
3. How data is stored
4. Their right to opt out
5. Human oversight process

### Consent Language Example

```
"ในการทำงานร่วมกัน ผมใช้ AI เป็นเครื่องมือช่วยวิเคราะห์ patterns
จากข้อมูลที่คุณให้ AI จะช่วยสรุปและหา patterns แต่จะไม่วินิจฉัย
หรือให้คำแนะนำทางคลินิก การตัดสินใจทั้งหมดยังเป็นของผม (ผู้เชี่ยวชาญ)
และคุณ (ผู้รับบริการ)

คุณยินยอมให้ใช้ AI ในการวิเคราะห์หรือไม่?"
```

## Privacy Considerations

### Data Protection

1. **Minimize data** — Collect only what's needed
2. **Anonymize** — Remove identifiers when possible
3. **Encrypt** — Protect data in transit and at rest
4. **Local first** — Prefer local processing
5. **Right to delete** — Honor deletion requests

### What to NEVER Share with AI

- Full names (use initials or codes)
- Contact information
- Financial data
- Exact addresses
- Other identifying details

## Implementation Checklist

### Before Using AI

- [ ] Define purpose clearly
- [ ] Get informed consent
- [ ] Establish data handling procedures
- [ ] Set up human oversight process
- [ ] Create documentation protocol

### During Use

- [ ] Monitor AI outputs
- [ ] Review before sharing with client
- [ ] Document AI involvement
- [ ] Maintain human relationship primacy

### After Use

- [ ] Evaluate effectiveness
- [ ] Address any concerns
- [ ] Update consent if needed
- [ ] Secure or delete data appropriately

## When to STOP Using AI

Stop immediately if:
- Client withdraws consent
- AI produces harmful output
- Clinical situation requires pure human attention
- Crisis situation
- Ethical concerns arise
