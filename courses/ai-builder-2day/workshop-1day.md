# Morning Session (9:00-12:00)

## "AI เป็นคู่คิด ไม่ใช่เครื่องมือ"

---

### 9:00-9:30 | Opening: The New Way (30 min)

**แนะนำตัว (10 min)**
- ทุกคนบอกชื่อ + ทำอะไร
- บอกปัญหาที่เตรียมมา (30 sec/คน)

**Demo สด (15 min)**
- แสดง: จากปัญหา → Prototype ใน 15 นาที
- ใช้ electrical calculator เป็นตัวอย่าง

**Message (5 min)**
> "นี่คือสิ่งที่คุณจะทำได้ตอนจบวันนี้"

---

### 9:30-10:30 | First Contact (60 min)

**Setup (10 min)**
- ทุกคนเปิด ChatGPT/Claude
- ทดสอบว่าใช้ได้

**Hands-on 1: ถามปัญหาของตัวเอง (30 min)**
```
1. พิมพ์ปัญหาของตัวเอง (ที่เตรียมมา)
2. ดู AI ตอบ
3. ถาม follow-up 2-3 รอบ
4. สังเกตว่า AI เข้าใจแค่ไหน
```

**Share (20 min)**
- แต่ละคนเล่าว่าได้อะไร
- ช่วยคนที่ติด
- สังเกต pattern ร่วม

---

### 10:30-10:45 | Break (15 min)

---

### 10:45-11:30 | Context is Everything (45 min)

**Concept (15 min)**

3 เรื่องสำคัญ:
1. **Input ดี = Output ดี** — ยิ่งให้ข้อมูลมาก ยิ่งได้คำตอบดี
2. **Upload ไฟล์ได้** — PDF, รูป, Excel, เสียง
3. **Iteration** — ถามได้หลายรอบ ไม่ต้องถูกตั้งแต่ครั้งแรก

**Demo (10 min)**
- ถามแบบไม่มี context → คำตอบกว้าง
- ถามแบบ upload file → คำตอบเฉพาะ
- ถามแบบมี custom instructions → คำตอบเหมาะกับคุณ

**Hands-on 2: ลอง upload ไฟล์ (20 min)**
```
1. Upload ไฟล์ตัวอย่างของตัวเอง
2. ถาม AI เกี่ยวกับไฟล์นั้น
3. ลองให้ AI สรุป/วิเคราะห์/แปล
4. สังเกตความแตกต่าง
```

---

### 11:30-12:00 | Define Your Project (30 min)

**Template ที่ทุกคนกรอก:**

```markdown
## My Project

**ปัญหา:** [1-2 ประโยค]

**ใครใช้:** [target user]

**Input:** [อะไรเข้า - file type, ข้อมูล]

**Output:** [อะไรออก - format, deliverable]

**ตัวอย่าง:**
- Input จริง: [attach ตัวอย่าง]
- Output ที่อยากได้: [อธิบาย]
```

**AI ช่วย Review (15 min)**
- ให้ AI ถามคำถาม clarify
- AI แนะนำว่าทำได้แค่ไหนใน 1 วัน
- ปรับ scope ถ้าจำเป็น

# Lunch (12:00-13:00)

# Afternoon Session (13:00-16:00/17:00)

## "ลงมือสร้าง"

---

### 13:00-14:30 | Build Session (90 min)

**เป้าหมาย:** ได้ version แรกที่ทำงานได้

**เลือก Track ตาม Background:**

#### Track A: No-Code (ChatGPT/Claude เปล่า)

```
1. สร้าง Custom GPT หรือ Claude Project
2. เขียน Instructions ที่ดี
3. ทดสอบกับ input จริง
4. ปรับปรุง prompt
5. ได้ "AI Assistant" ของตัวเอง
```

**Output:** Custom GPT หรือ Claude Project ที่ทำงานได้

#### Track B: Low-Code (ใช้ tools ช่วย)

```
1. เลือก tool: Zapier, Make, n8n
2. AI ช่วยออกแบบ workflow
3. ต่อ integration
4. ทดสอบ
5. ได้ automation ที่ทำงานได้
```

**Output:** Workflow ที่ทำงานอัตโนมัติ

#### Track C: With Code (มี technical background)

```
1. AI ช่วย generate code
2. Setup project (ใช้ Replit/Vercel)
3. Deploy
4. ทดสอบ
5. ได้ web app
```

**Output:** Working prototype deployed

**Instructor Role:**
- เดินดูแต่ละคน
- ช่วยคนที่ติด
- Pair คนเก่งกับคนใหม่

---

### 14:30-14:45 | Break (15 min)

---

### 14:45-15:30 | Iterate & Improve (45 min)

**Flow:**
```
1. ทดสอบกับ input จริง
2. หา bug / สิ่งที่ขาด
3. บอก AI ให้ช่วยแก้
4. ทดสอบอีกรอบ
5. Repeat
```

**Key Teaching:**
> "Development คือ loop: Build → Test → Fix → Repeat"
> "AI ทำให้ loop นี้เร็วมาก"

---

### 15:30-16:00 | Show & Tell (30 min)

**ทุกคนแชร์ (3-5 นาที/คน):**
1. ปัญหาที่แก้ (30 sec)
2. **Demo ของจริง** (2 min)
3. สิ่งที่เรียนรู้ (1 min)
4. Q&A (1 min)

**Instructor:**
- ถ่ายวิดีโอ (ให้ผู้เรียนเก็บ)
- Celebrate ทุกคน

---

### 16:00-16:30 | Closing: What's Next (30 min)

**สรุป (10 min)**
1. วันนี้ได้อะไร
2. ทำต่อยังไง
3. Resources

**Tips for Independence (10 min)**

```
1. Custom Instructions — บอก AI ว่าคุณเป็นใคร
2. Save & Organize — แยก chat ตาม project
3. Iteration — ไม่ต้องถูกครั้งแรก
4. Community — ถามกันได้
```

**แจก (5 min)**
- Resource list (links, docs)
- Group chat access
- Follow-up support info

**Q&A (5 min)**

---

## Optional: Extended Session (16:30-17:00)

## Key Principles

1. **Learn by doing** — 80% hands-on
2. **Real problems** — ไม่ใช่ตัวอย่างสมมติ
3. **Finish something** — ต้องได้ของจริง
4. **Independence** — ต้องทำต่อเองได้
