# Workshop: Metaphor Over Technical Terms

**Date**: 2025-12-22

## What We Learned

- **Metaphor first, technical later** — "ลูกน้อง" เข้าใจทันที, "Subagent" ต้องอธิบาย
- **Live coding > slides** — สร้างให้ดู 2 นาที ดีกว่าอธิบาย 20 นาที
- **Personal demo memorable** — "หาเรื่องเมา" จำได้, "search git history" ลืม
- **Decision tree สำคัญ** — ไม่ใช่แค่ "คืออะไร" แต่ "ใช้เมื่อไหร่"

## The Pattern

| Technical | Metaphor | Why It Works |
|-----------|----------|--------------|
| Subagent | ลูกน้อง | ทำงานแยก, มี boss |
| Command | ปุ่มลัด | พิมพ์แล้วเกิด action |
| Skill | คู่มือ | อ่านเองเมื่อต้องใช้ |

## How Things Connect

- **rrr session** → realized memory search demo worked best
- **tong-finder creation** → proved live coding = "แค่นี้เอง?"
- **Slides generation** → 10-slide structure emerged from teaching flow

## Apply When

- Teaching feature-rich tools to beginners
- Workshop prep — start with metaphor, end with technical
- Demo selection — pick personal, memorable examples
