## Key Topics

### 1. What is MAW
- Multi-Agent Workflow = tmux + git worktrees orchestration
- Each agent = isolated directory with own branch
- File signals for fast communication (~100ms)
- Origin: `Soul-Brews-Studio/multi-agent-workflow-kit`
- See: **Comprehensive Guide § 1**

### 2. Architecture
```
repo/
├── agents/
│   ├── 1/ (branch: agents/1)
│   ├── 2/ (branch: agents/2)
│   ├── 3/ (branch: agents/3)
│   ├── 4/ (branch: agents/4)
│   └── 5/ (branch: agents/5)
└── .agents/
    ├── profiles/      ← Different layouts (profile0, profile14, etc.)
    ├── scripts/       ← maw-wrapper, start-agents, etc.
    └── config.sh      ← MAW configuration
```
- See: **Comprehensive Guide § 2**

### 3. Installation
- `maw init [profile]` — Initialize with layout
- Copy profiles from other repos if needed
- Ensure scripts match (especially start-agents.sh for layout support)
- See: **Setup Patterns** + **Copy Order Pattern**

### 4. Profiles & Layouts
- **profile0** = Vertical layout (1 window × N panes)
- **profile14** = Horizontal layout (N windows × 3 panes each) ← Current favorite
- Profiles are in `.agents/profiles/profile-{N}.json`
- See: **Setup Mistakes § Profile Confusion**

### 5. Control Commands
- `maw start [agents]` — Launch agents in tmux
- `maw hey [target] [msg]` — Send signal to agent(s)
- `maw kill` — Stop all agents (WARNING: has y/N prompt, can hang)
  - Better: Use `tmux kill-session` directly
- See: **Start & Control Patterns**

### 6. Golden Rules
1. Never force kill — use tmux kill-session if maw kill hangs
2. Profile scripts must match layout (profile0 vs profile14)
3. Python version matters (3.14 not compatible, use 3.12)
4. Real paths matter in control scripts
5. File signals are reliable fast comms

### 7. Pain Points & Solutions
| Problem | Solution |
|---------|----------|
| `maw kill` hangs | Use `tmux kill-session` directly |
| Wrong layout | Check start-agents.sh has layout logic |
| Python 3.14 errors | Use Python 3.12 |
| Profile mismatch | Copy complete profile directory from reference repo |
| Path errors | Use $PWD, $(cd ...) patterns, not hardcoded |
