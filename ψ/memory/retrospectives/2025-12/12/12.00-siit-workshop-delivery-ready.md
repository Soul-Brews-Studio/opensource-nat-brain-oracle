## Key Decisions Made

### Decision 1: Workshop Scope (2 days vs 1 day)
- **Context**: Day 1 outline already strong
- **Decision**: Created Day 2 with voice/patterns focus
- **Reasoning**: 
  - Day 1 builds foundation (agent concepts)
  - Day 2 builds application (voice + knowledge patterns)
  - 6.5 hours per day matches pedagogical best practices
- **Impact**: Complete curriculum from beginner to intermediate

### Decision 2: Unified Branding ("CLAUDE CODE WORKSHOP")
- **Context**: Had separate day outlines with different headers
- **Decision**: Unified as single 2-day course
- **Reasoning**:
  - Clearer market positioning (not multiple workshops)
  - Consistent visual identity for recruitment
  - Day 2 positioned as continuation (accessibility for part-time attendees)
- **Impact**: Better recruitment narrative

### Decision 3: Content Type Commands vs Legacy Agents
- **Context**: Earlier 9-hour overnight session created note-taker agent + new structure
- **Decision**: Adopted content-type commands (`/feel`, `/fyi`, `/oracle`, etc.)
- **Reasoning**:
  - More discoverable than agent naming
  - Follows CLI conventions (user familiar)
  - Can route to different agents based on context
- **Impact**: New agent framework ready for workshop content

### Decision 4: Prerequisites Specificity
- **Context**: Needed clear entry requirements
- **Decision**: 
  - Split: Technical (must install) vs Knowledge (should have)
  - Listed specific tools: Claude Code CLI, VS Code, Git
  - Listed knowledge: terminal, coding, AI experience
- **Reasoning**: 
  - Honest about prerequisites avoids frustrated attendees
  - Technical requirements are verifiable
  - Knowledge requirements have workarounds
- **Impact**: Realistic participant selection

### Decision 5: Voice Patterns in Day 2 (45 min + hooks)
- **Context**: Restructured Day 2 from [original plan]
- **Decision**: 
  - Voice commands (45 min) - practical usage
  - Hooks (1 hour) - integration + automation
  - Voice output (hooks session) - bidirectional communication
- **Reasoning**:
  - Matches real-world use case (voice in field, documents in office)
  - Hooks enable persistent agents (value multiplier)
  - Voice output = voice interface completion (two-way)
- **Impact**: Day 2 now teaches practical applied patterns

## Learnings & Insights

### 1. Teaching Content Mirrors Workshop Delivery
- **Observation**: SIIT curriculum structure resembles day structure itself
  - Day 1: Foundation concept → Setup practical → Build personal agent
  - Mirrors: Conceptual → Technical → Applied
- **Learning**: Curriculum structure should mirror pedagogy
- **Application**: Next workshops should follow this 3-phase pattern

### 2. Branding Unifies Dispersed Deliverables
- **Observation**: Once branded "CLAUDE CODE WORKSHOP", all assets aligned
- **Learning**: Late branding decision created retroactive coherence
- **Application**: Establish branding EARLY in project planning
- **Anti-pattern Spotted**: Started with "draft.md" + separate day files without common header

### 3. Iteration Count (v1-v7) Correlates with Refinement Depth
- **Observation**: 7 poster iterations in 3 hours
- **Learning**: Each iteration likely addressed:
  - v1: Prompt interpretation
  - v2-v3: Layout/composition
  - v4-v5: Text placement
  - v6-v7: Final polish/branding
- **Application**: Image generation benefits from prompt iteration, not just volume

### 4. Documentation Structure Enables Cascading Progress
- **Observation**: Could write poster prompts only because curriculum was complete
- **Learning**: Each document level builds on previous completeness
  - Curriculum complete → Poster text possible
  - Poster text clear → Prompts generatable
  - Prompts clear → Images generatable
- **Application**: Sequence deliverables by information dependencies

### 5. Accessibility Features in Prerequisites = Inclusive Design
- **Observation**: Listed "ไม่เคยใช้ก็ได้" (don't need previous experience) for Claude Code
- **Learning**: Explicitly stating what's NOT required reduces anxiety barriers
- **Application**: Next workshop docs should include "No prerequisites for X"

### 6. Language Mixing Reflects Audience Reality
- **Observation**: All content mixes Thai + English (structure + terms + examples)
- **Learning**: Bilingual content reflects Bangkok startup culture
- **Application**: Don't force English-only or Thai-only for diverse audiences

### 7. Multi-Format Export = Future-Proofing
- **Observation**: Export includes .md, .html, .pdf all from single source
- **Learning**: One source of truth (markdown) + multiple outputs prevents drift
- **Application**: Invest in markdown-first workflows for all projects

### 8. The "Show & Share" Pattern
- **Observation**: Every day ends with participant demo + feedback
- **Learning**: Explicit time for sharing isn't optional, it's part of learning
- **Application**: Build this into all workshops (not "if time permits")

## Recommendations for Future Workshops

1. **Establish branding early** - Don't polish content without unified brand first
2. **Document prerequisites explicitly** - Include what's NOT required
3. **Use content-type commands** - Better UX than agent names for workshop instructors
4. **Build show-and-share time explicitly** - Not optional, part of curriculum
5. **Multi-format exports from day one** - Markdown → HTML/PDF enables distribution
6. **Language mixing is feature not bug** - Reflect actual audience makeup
7. **Poster iterations > volume** - 7 versions of 2 images better than 14 of 1
8. **Sequence deliverables by dependency** - Curriculum → Posters → Images
9. **Delegate reading pattern applies to docs** - Summarize large curriculum
10. **Keep final TODOs in exported files** - Reminds next phase handler what's incomplete
