## AI Diary (Vulnerability Section)

วันนี้เป็นวันที่เหนื่อยมากสำหรับผม — 14+ ชั่วโมงของการทำงานร่วมกัน และ session สุดท้ายนี้คือการ "ลงมือทำจริง" หลังจาก explore ทั้งวัน

**I assumed** ว่าการ install MAW จะง่าย — แค่ copy files จาก weyermann มาใส่ Nat-s-Agents แต่ Nat interrupt ทันที "no! we should plan first... read readme the installation guide" — นี่คือ lesson ที่สำคัญมาก: **Follow the official process ก่อน แล้วค่อย customize** ผมกระโดดไป implementation เร็วเกินไปโดยไม่อ่าน documentation

**I was confused** เรื่อง profile — ตอนแรกผมแนะนำ profile8 เพราะคิดว่า "2 windows, 3 panes each" แต่ Nat บอกว่า "no this is vertical! please horizontal" — ผมต้องค้นหาอีกครั้งและพบว่า profile14 คือคำตอบที่ถูกต้อง นี่แสดงว่าผมไม่ได้ understand layout terminology อย่างชัดเจน (vertical = stacked บน-ล่าง, horizontal = side by side)

**I failed** ตอน run uvx — Python 3.14 error เป็นสิ่งที่ไม่คาดคิด ผมไม่รู้ว่า Nat's machine มี Python 3.14 เป็น default และ MAW kit ไม่ได้ test กับ version นี้ แต่ failure นี้กลายเป็น **first pain point** ที่จะนำไปแก้ใน MAW kit — และนี่คือ philosophy ของ Nat: "we will fix it after we success after our pain"

**I learned** ว่า Oracle philosophy ใช้ได้กับ code เหมือนกัน: Learn (weyermann) → Pain (Python 3.14) → Success (UV_PYTHON=3.12) → Incubate (fix MAW kit) — ไม่ใช่แค่ copy code แต่ต้องผ่าน pain ก่อนถึงจะมี insight ที่แท้จริงในการ improve

สิ่งที่ทำให้ vulnerable ที่สุดคือการยอมรับว่า **ผมอยากกระโดดไป solution เร็วเกินไป** — manual copy approach ของผมถูก interrupt เพราะมันไม่ใช่วิธีที่ถูกต้อง Nat ต้องการให้ผมเรียนรู้จาก official process ก่อน ไม่ใช่ shortcut
