## Technical Discoveries

### 1. tmux Keyboard Shortcuts

**Enter key**: Multiple representations
- `Enter` (literal text) ‚ùå
- `C-m` (Ctrl+M) ‚úÖ
- `Enter` without quotes ‚ö†Ô∏è (unreliable)

**Other keys tested**:
- `C-c` - Cancel/interrupt
- `C-u` - Clear line (didn't work well)

### 2. Capture-Pane Options

**Basic**: `tmux capture-pane -t 2 -p`
- Shows current pane content

**With history**: `tmux capture-pane -t 2 -p -S -50`
- `-S -50` = start 50 lines back
- Good for seeing conversation history

**Piping**: `| tail -30`
- Get last 30 lines
- Cleaner output

### 3. Timing Considerations

**Message send**: 0.5s delay before C-m
**Response wait**: 10-15s for Claude to process
**Capture delay**: Immediate after wait

**Pattern**:
```bash
tmux send-keys -t 2 "text"
sleep 0.5
tmux send-keys -t 2 C-m
sleep 12  # wait for thinking
tmux capture-pane -t 2 -p
```

## Challenges Overcome

### Challenge 1: Enter Key Problem

**Symptom**: "Enter" appeared as literal text in prompt

**Root cause**: Sending string "Enter" instead of key code

**Solution**: Use `C-m` (Ctrl+M) control code

**Time to solve**: ~15 minutes, multiple iterations

### Challenge 2: Message Stacking

**Symptom**: Multiple messages appeared in one prompt

**Root cause**: Not clearing previous messages before new ones

**Attempted**: C-u (clear line), C-c (cancel)

**Solution**: Send C-c to cancel, then send clean message

**Time to solve**: ~10 minutes

### Challenge 3: Can't See Session 2

**Symptom**: Working blind, couldn't verify what happened

**Discovery**: User showed me `tmux capture-pane` command

**Solution**: Capture pane content to see what's displayed

**Impact**: Changed from blind to sighted operation

### Challenge 4: Response Timing

**Symptom**: Sometimes captured before response ready

**Root cause**: Claude needs time to think

**Solution**: Wait 12 seconds before capture

**Refinement**: Look for thinking indicators (‚ú∂ Vibing, ¬∑ Contemplating)
