## Key Insight: Beads vs Nat-s-Agents

| | Beads | Nat-s-Agents |
|---|---|---|
| **Core Question** | "What's blocking this task?" | "Why did I decide this?" |
| **Metaphor** | Task graph database | Extended consciousness |
| **Storage** | JSONL + SQLite | Markdown + Git |
| **Best For** | Multi-agent coordination | Solo dev knowledge |

**Complementary, not competing!**
- Beads = task coordination (dependency graph)
- Nat-s-Agents = knowledge synthesis (append-only memory)

## Beads Highlights

Steve Yegge's design choices:
- **Hash-based IDs** (`bd-a1b2`) - zero merge conflicts
- **Daemon + direct mode** - RPC for speed, SQLite fallback
- **Content hashing** - SHA256 for deduplication
- **`--json` everywhere** - agent-first design
