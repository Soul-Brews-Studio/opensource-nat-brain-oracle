## Honest Feedback

### What Could Be Better
1. **Session continuity** - Reading compacted summaries takes time to orient. The summary was good but I still needed to re-read files to understand state.

2. **Voice notification architecture** - Currently only announces when Logs tab is open. User might want background notifications. Mentioned this but didn't implement.

3. **Testing** - Each voice change required manual testing. No automated way to verify MQTT messages are sent correctly.

### Friction Points
- Compacted context means some history is lost
- Multiple background tasks running made output harder to track
- Voice settings are hardcoded - could be user-configurable in UI
