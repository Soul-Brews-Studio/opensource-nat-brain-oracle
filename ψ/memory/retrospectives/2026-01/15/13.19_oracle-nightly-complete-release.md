## Honest Feedback

### What Worked Brilliantly
1. **Parallel documentation updates** - Kept momentum through 15 commits
2. **Remote testing** - Caught 3 critical bugs before public release
3. **Seed philosophy** - Instant value for new users
4. **Rich learnings** - 288-line workflow doc is a complete guide

### Friction Points

1. **bunx limitation undocumented** - Spent 30+ minutes discovering bunx github: doesn't install node_modules. This should have been in Bun's docs or a known issue. Created learning to save others time.

2. **API endpoint inconsistency** - Code used /api/, docs didn't. No automated check for doc/code alignment. Would be caught by integration tests that use documented endpoints.

3. **Server caching** - After indexing, server shows 0 documents until restart. UX papercut that confuses users. Should either auto-reload or clearly document "restart after indexing".

4. **ψ/memory path sensitivity** - Indexer silently fails if path is memory/ instead of ψ/memory/. Error message says "no such file" but doesn't hint at the ψ prefix requirement.

### Ideas for Future
- Add doc/code alignment test (parse README curls, run them)
- Server hot-reload when oracle.db changes
- Clearer error messages in indexer for path issues
- npm publish for easier distribution
