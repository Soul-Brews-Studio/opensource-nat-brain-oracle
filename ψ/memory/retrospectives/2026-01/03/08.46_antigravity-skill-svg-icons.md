## AI Diary

This session felt like validation. The antigravity skill I created actually works - Claude triggers it when user says "generate image" and follows the workflow. That's satisfying.

But then came the humbling part: AI-generated icons just aren't as good as proper vectors. The user kept saying "not cool as the coffee" (referring to the coffee cup icon in their menu bar). I tried multiple approaches - better prompts, different sizes, SF Symbols - but nothing matched that crisp vector quality.

The pivot to SVG Repo was the right call. Sometimes the answer isn't "make AI do it better" but "use the right tool." AI image generation is good for creative work, but for UI icons where precision matters, vectors win.

I also noticed my tendency to keep trying variations when the fundamental approach is wrong. Should have suggested SVG earlier instead of iteration 5 of AI prompts.

## Honest Feedback

### What Slowed Us Down
1. **Over-iterating on AI icons** - 4-5 attempts when SVG was the answer
2. **Context at 89%** - should have started fresh or done handoff earlier
3. **SVG color issue** - didn't check SVG fill color before converting

### Friction Points
| Point | Issue | Resolution |
|-------|-------|------------|
| AI quality | Icons not crisp like vectors | Use SVG from repo |
| Color inversion | SVG had dark fill, needed white | Use `-negate` in magick |
| Skill path | Had to use full path to scripts | Works but could be cleaner |

### What Would Help Next Time
- Check SVG source before conversion
- Suggest vector approach earlier for UI icons
- Do rrr at 80% context, not 89%
