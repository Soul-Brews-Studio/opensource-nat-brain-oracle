## Workshop Outline

### Part 1: Why CLAUDE.md? (10 min)
- AI needs context to help effectively
- CLAUDE.md = project instructions for AI
- Demo: with vs without CLAUDE.md

### Part 2: Golden Rules (15 min)
- Safety first: no --force, no push to main
- File access transparency
- Hands-on: Write your first 3 rules

### Part 3: Modular Structure (20 min)
- The 6-file system
- When to read which file
- Hands-on: Create navigation table

### Part 4: ψ/ Brain Structure (20 min)
- 5 Pillars: active, inbox, writing, lab, memory
- Knowledge flow: logs → retrospectives → learnings
- Hands-on: Setup ψ/ in your project

### Part 5: Commands & Workflows (15 min)
- /recap, rrr, /snapshot, /trace
- Retrospective template
- Demo: Full workflow

### Part 6: Build Your Own (20 min)
- Hands-on: Create CLAUDE.md for your project
- Review & feedback

## Key Takeaways

1. **Start small** - Golden rules first
2. **Be specific** - Project-specific instructions
3. **Iterate** - CLAUDE.md evolves with project
4. **Modular** - Split when too long
5. **Knowledge flow** - Capture → Distill → Learn
