## Structure

```
active/
├── INDEX.md              ← You are here
├── [task-name].md        ← Current task plan
└── context/              ← Temp files (delete when done)
```

## Workflow

```
1. รับ task → สร้าง active/[task].md
2. ทำงาน → temp files ใน context/
3. เสร็จ → ย้ายไป memory/ หรือลบ
```

## Rules

- `context/` ควรว่างเปล่า
- ไม่สะสม research เก่า
- Plan file = single source of truth
